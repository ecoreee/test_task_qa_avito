## Тест-кейсы для проверки api микросервиса
### 1. Создание объявления:
#### Тест-кейс 1: Успешное создание объявления с валидными данными
+ Эндпоинт: POST /api/1/item
+ Тестовые данные: 
{
"sellerId": 123456,
"name": "Test item",
"price": 1000,
"statistics": {"likes": 5, "viewCount": 100, "contacts": 3}
}
+ Шаги: 
  1. Отправить POST запрос с валидными данными
  2. Получить ответ от сервера
+ Ожидаемый результат:
  1. Status Code: 200
  2. Response Body содержит поля: id, sellerId, name, price, statistics, createdAt
  3. id - строка
#### Тест-кейс 2: Создание объекта с невалидными данными
+ Эндпоинт: POST /api/1/item
+ Тестовые данные: 
{
"sellerId": "invalid_str",
"name": "Test item",
"price": "not_num",
}
+ Шаги: 
  1. Отправить POST запрос с невалидными данными
  2. Получить ответ от сервера
+ Ожидаемый результат:
  1. Status Code: 400
  2. Response Body содержит сообщение об ошибке

### 2. Получение объявления по ID 
#### Тест-кейс 5: Успешное получение существующего объявления
+ Эндпоинт: GET /api/1/item/{id}
+ Тестовые данные: Нет
+ Шаги: 
  1. Создать объявление и запомнить ID, 
  2. Отправить GET запрос к /api/1/item/{id}
+ Ожидаемый результат:
  1. Status Code: 200
  2. Response Body содержит содержит массив с данными объявления
  3. Данные соответсвуют созданному объявлению
